# Evidence Gap Taxonomy and Review Lanes

> [!IMPORTANT]
> **Prototype System Notice**  
> Caesar AI Scan is an offline static-analysis prototype. It identifies governance review needs, not final legal conclusions. The findings generated are AI usage signals, not definitive proof of non-compliance. All evidence export candidates generated by the exporter remain in a `draft` status and require human review and cryptographic authorization before any official Caesar AI Evidence or Governance OS ingestion occurs.

This document describes the structured governance taxonomy, assigned review lanes, and classified evidence gaps defined in `data/review-taxonomy.ai-governance.json` and `data/evidence-requirements.ai-usage.json`.

## 🗺️ Governance Review Lanes

Review lanes define the cross-functional corporate domains responsible for signing off on different AI usage characteristics:

| Lane Key | Lane Name | Target Focus |
| :--- | :--- | :--- |
| `technical_owner_review` | Technical Owner Review | Code boundaries, operational dependencies, ownership assignment. |
| `security_review` | Security Review | plain-text credentials, vulnerabilities, secret key exposure. |
| `legal_compliance_review` | Legal Compliance Review | Third-party model licensing, regulatory alignment (e.g., EU AI Act). |
| `privacy_review` | Privacy Review | PII leakage risks, vector indexing, user consent pipelines. |
| `vendor_review` | Vendor Review | SaaS provider SLAs, DPA terms, and processing jurisdictions. |
| `model_inventory_review` | Model Inventory Review | Tracking model names, versions, weights provenance, and catalogs. |
| `risk_classification_review` | Risk Classification Review | Categorizing AI application risk level (Minimal, Limited, High, Unacceptable). |
| `human_oversight_review` | Human Oversight Review | Checking human-in-the-loop validation, override interfaces, and guardrails. |
| `retention_and_logging_review`| Retention & Logging Review | Purging schedules and audit logs for prompts and completions. |

## 🛑 Classified Evidence Gaps

Evidence gaps track missing corporate governance elements. In this offline prototype, every candidate requires manual promotion because multiple gaps remain unresolved:

| Gap Type | Default Severity | Blocks Export | Required For | Description / Recommended Resolution |
| :--- | :---: | :---: | :--- | :--- |
| `missing_system_owner` | Medium | Yes | `technical_owner_review` | No technical or business owner is assigned. Assign a lead software engineer or system owner. |
| `missing_processing_purpose` | Medium | Yes | `legal_compliance_review` | The operational purpose is undefined. Document the concrete business goals. |
| `missing_ai_use_case_description` | Low | No | `model_inventory_review` | No usage description is associated. Add a brief system summary. |
| `missing_model_or_provider_identification` | High | Yes | `model_inventory_review` | The exact model name or provider is missing. Configure specific model variant. |
| `missing_vendor_terms_review` | High | Yes | `vendor_review` | No verified vendor DPA or commercial SLA. Review provider terms to verify training exclusion. |
| `missing_data_processing_assessment` | High | Yes | `privacy_review` | No DPIA has been completed. Conduct a DPIA to evaluate risks. |
| `missing_risk_classification` | High | Yes | `risk_classification_review` | Integration lacks a formal risk tier. Complete risk assessment survey. |
| `missing_human_oversight_description` | Medium | No | `human_oversight_review` | No human-in-the-loop description. Document validation gates and fallbacks. |
| `missing_logging_or_retention_policy` | Medium | No | `retention_and_logging_review`| Logs lack a clear purging policy. Define log purges and retention intervals. |
| `missing_user_disclosure_assessment` | Low | No | `legal_compliance_review` | Lacks user-facing generative AI interaction notices. Verify disclosure presence. |
| `missing_security_review` | High | Yes | `security_review` | plaintext credentials exposed in configuration. Migrate env credentials to secrets manager. |
| `missing_export_authorization` | Medium | Yes | `technical_owner_review` | Lacks cryptographic lead signature. Acquire cryptographical signature in portal. |
