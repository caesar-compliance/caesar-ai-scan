# Scan Diff Product View

## Overview
The Scan Diff Product View is a local-only governance insight layer that highlights changes between consecutive AI governance scans. It provides product-centric summaries of added, removed, and changed AI usage signals, helping auditors track compliance drift over time.

## Key Features
- **Summary Cards**: Quick count of new, removed, and changed AI signals.
- **Signals Impact**: Detailed list of newly introduced AI dependencies.
- **Governance Implications**: Automated analysis of how changes affect the overall AI risk profile.
- **Ecosystem Tracking**: Aggregated view of affected AI providers (OpenAI, Anthropic, etc.) and frameworks (LangChain, etc.).

## Local Data Sources
The view is generated by `scripts/build-scan-diff-product-view.mjs`, which utilizes:
1. `tmp/sample-history/latest-diff.json`: Raw diff data generated by the scanner's history module.
2. `src/scan-diff-product/scan-diff-product-view-report.mjs`: Enrichment logic to categorize raw findings.

If no local history is available, the builder falls back to a deterministic `sample_only` mode.

## How to Build & Validate
To generate the product view data:
```bash
npm run scan-diff:product-view
```

To validate the generated JSON against its schema:
```bash
npm run validate:scan-diff-product-view
```

## Governance Value
- **Continuous Monitoring Rehearsal**: Simulates a "what changed?" workflow essential for high-frequency CI/CD governance.
- **Risk Drift Identification**: Alerts auditors when new AI capabilities (e.g., a new LLM provider) are introduced without prior review.

## Limitations
- **Local Static Data**: The view reflects a point-in-time comparison of local files.
- **No Live Monitoring**: It does not track real-time API calls or runtime behavior.
- **Backend Disabled**: No connection to live databases or cloud services is used.
