# Evidence Export Pack Reference Guide

This document describes the design, structure, and usage of the self-contained offline **Evidence Export Pack** compiled by `caesar-ai-scan` version `0.4.0`.

---

## 📖 Overview

The **Evidence Export Pack** (`CaesarEvidenceExportPack`) is a local-first, zero-dependency audit package created during a static analysis codebase scan. It acts as an intermediate staging bundle for manual sign-off by a compliance officer before the data is ingested downstream into **Caesar AI Evidence** or the **Governance OS**.

To support completely offline and air-gapped operations, the package bundles all required schemas, scan results, evidence candidate drafts, review workflows, compliance gaps, check checklists, and cryptographic hashes into a single directory.

---

## 📂 Package Directory Structure

When running the scanner with the `--export-pack <directory>` parameter, the following 8 files are written:

| Filename | Type | Role | Description |
| :--- | :--- | :--- | :--- |
| **`export-pack.json`** | JSON | Parent Index | The master document compiling all scan, review, readiness, checklist, and manifest objects in-memory. |
| **`manifest.json`** | JSON | Integrity Tracker | Maps files, record counts, and registers standard SHA-256 cryptographic hashes for integrity. |
| **`scan-result.json`** | JSON | Scan Findings | The raw static analysis run output listing all identified AI components and credentials. |
| **`evidence-candidates.json`** | JSON | Export Candidates | Interoperable draft candidate records mapped to Caesar AI Evidence registers. |
| **`review-workflow.json`** | JSON | Review Workflow | The compiled review items mapping findings to assigned review lanes and evidence gaps. |
| **`import-readiness.json`** | JSON | Readiness Feasibility | Calculates readiness scores, blocking gaps, and locks automated ingestion. |
| **`human-review-checklist.json`** | JSON | Auditor Checklist | Compiles auditor verification questions, signature checkboxes, and routing lanes. |
| **`REVIEW_SUMMARY.md`** | Markdown | Auditor Report | A visual compliance dashboard highlighting metadata, readiness scores, blockers, and manual sign-offs. |

## 🔒 Policy Boundaries & Isolation

To prevent accidental compliance bypasses or data leakages, the export pack enforces two strict policy boundaries:

1. **Draft Hardlock Boundary (Policy A):** All candidates in `evidence-candidates.json` are hardcoded to `"status": "draft"` with `"review_required": true`. Approval state changes cannot be made automatically or programmatically during the static analysis scan.
2. **Ingestion Isolation Boundary (Policy B):** The field `import_readiness.can_import_automatically` is locked to `false`, and `human_review_checklist.signoff_required` is locked to `true`. This guarantees that human review remains a strict gateway requirement for all scanned codebase assets.
